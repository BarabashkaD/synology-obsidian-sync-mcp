# Contributing

All changes must be made through pull requests.

1. Create a feature branch from `main`.
2. Keep secrets, generated `.env` files and runtime data out of Git.
3. Run `bash -n create-obsidian-user-stack.sh` before opening a pull request.
4. Run ShellCheck when available.
5. Explain Synology DSM assumptions and security implications in the pull request.

Direct commits to `main` are intentionally blocked by the repository ruleset.
